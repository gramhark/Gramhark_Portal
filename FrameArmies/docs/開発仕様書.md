# ゲーム基本仕様書（Ver 2.0）

## 1. プロジェクト概要

| 項目 | 内容 |
| :--- | :--- |
| ターゲット層 | 誰でも |
| プラットフォーム | Webブラウザゲーム（HTML5 / JavaScript / CSS3） |
| コンセプト | 「デッキ構築」×「配置型タワーディフェンス」×「パズル」 |
| アートテーマ | フレームの中が戦場。外側がプレイヤーの操作エリア。 |
| デザインコンセプト | 「クラシック・ミニマル・スタイリッシュ」。漆黒（#000000）の背景に、細い白線と鮮やかな画像が映える、無駄を削ぎ落としたインターフェース。 |
| 使用技術 | Phaser 3（ゲームエンジン）、Howler.js（音声処理）、HTML5 / CSS3 |

---

## 2. 画面レイアウト

- **画角：16:9固定。** PCはそのまま横画面、モバイルは横画面でプレイ。
- **メインフィールド（額縁の中）:** 8×8のグリッド。
- **プレイヤーエリア（額縁の外）:**
  - **ステータス表示:** 総コスト（現在値 / 15）、デッキ残り枚数、現在のターン数。
  - **手札エリア:** 常に3枚のカードが表示される。
  - **操作ボタン:** 「アクション開始（Go）」「パネル撤去モード」等。

---

## 3. 基本ルールと進行方式

- **基本システム:** ターン制。
- **ドローと手札:** 初期手札は3枚。毎ターン開始時に山札から1枚引き、常に手札が3枚になるよう維持される。デッキは全20枚。
- **配置制限（コストシステム）:**
  - 盤面に出せるパネルの「総コスト」は最大15まで。
  - 上限に達している場合、新しいパネルを置くには、配置済みのパネルを選んで「撤去」する必要がある。
- **勝利・敗北条件:**
  - 勝利：全ウェーブのモンスターを撃破する。
  - 敗北：モンスターに最下部を突破される、またはデッキを使い切り全滅させられない。
  - デッキ残り枚数は残量に応じて表示色が変化する（通常：白 → 残り10枚以下：黄色 → 残り5枚以下：赤）。

---

## 4. 武器パネル（デッキ構成要素）

※全ての武器の基本攻撃力は「1」とする。キャラクターは画像で表示。  
※🔮魔法のみ範囲内の敵全員に攻撃。他のパネルは設置時に指定した1方向にのみ攻撃する。

| 名称 | 攻撃範囲 | コスト | 特徴 |
| :--- | :--- | :--- | :--- |
| 剣 | 上下左右1マス | 1 | 低コストで扱いやすい基本兵装。設置時に向きを指定。 |
| 弓 | 上下左右2マス先 | 2 | 離れた敵を狙えるが、隣接されると隙ができる。設置時に向きを指定。 |
| 魔法 | 周囲3×3マス | 4 | 広範囲の敵全員を一掃できるが、コストが高い。 |
| 氷 | 上下左右1マス | 2 | ダメージなし。敵を1ターン移動停止させる。複数の氷が同じ敵に当たっても停止は1ターン固定（上書き）。設置時に向きを指定。 |
| 盾 | 自身のマス | 2 | 攻撃力0（特例）。敵の進行を完全に1回防いで壊れる。コウモリは飛び越え、ゴースト・ドラゴンには無効。 |
| 爆弾 | 周囲3×3マス | 3 | 設置2ターン後に爆発（ダメージ3）。範囲内全員にダメージ。使い捨て。爆発までのカウントダウンを視覚表示する。 |

---

## 5. パネルとモンスターの接触ルール

- **盾以外のパネル:** モンスターが同じマスに進入した瞬間にパネルが破壊される。
- **盾:** モンスターの進入を完全に防ぐ（1回防いで壊れる）。

---

## 6. 武器パネルの攻撃タイミング

- 毎ターン、各パネルの攻撃範囲内に敵がいるかを判定する。
- 敵がいる場合、そのターンに1回だけ攻撃する。
- 魔法のみ範囲内の敵全員にダメージ。他のパネルは設置時に指定した1方向の敵のみ攻撃する。

---

## 7. モンスター（敵キャラクター）

※HP1の敵は1回の攻撃で倒せる。キャラクターは画像で表示。

| 名称 | HP | 移動パターン | 特徴 |
| :--- | :--- | :--- | :--- |
| スライム | 1 | 毎ターン1マス下 | 基本的な敵。 |
| コウモリ | 1 | 毎ターン斜め下 | 盾を飛び越えて移動する。 |
| イノシシ | 2 | 毎ターン2マス下 | 移動が速く、1発では倒れない強敵。 |
| ゴースト | 1 | 毎ターン1マス下 | 剣・弓を50%で回避。魔法は必中。 |
| ドラゴン | 5 | 2ターンに1マス下 | 圧倒的な耐久力。毎ターン隣接しているパネルを1つ破壊する。 |

---

## 8. 敵の出現（ウェーブ）ルール

- **スポーン地点:** 最上段（行0）のランダムな列から出現。
- **出現ペース:**
  - 序盤（1-5T）：1ターンに1体出現。
  - 中盤（6-15T）：1ターンに1-2体出現。イノシシ等の混成。
  - 終盤（16T-）：ドラゴンが登場し、出現密度が上がる。
- プレイヤーは次に出現する敵の「予兆（影やマーク）」を最上段で確認できる。

---

## 9. ビジュアル・デザイン指針

- **背景:** 深い黒（#000000）のDark Mode。
- **グリッド:** 非常に細いグレーの線。キャラクターがいないマスは無表記。
- **フォント:** 数字やテキストは視認性の高いサンセリフ体（英字はモダンな等幅フォント）を使用。
- **アニメーション・エフェクト（Phaser 3使用）:**
  - 攻撃時：パネルが少し震える、または火花が散るミニマルな演出。
  - ダメージ時：敵が一瞬半透明になる。
  - 爆弾：爆発までのカウントダウンを視覚的に表示する。
- **デッキ残量の色変化:**
  - 通常：白
  - 残り10枚以下：黄色
  - 残り5枚以下：赤
